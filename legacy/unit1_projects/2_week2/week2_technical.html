CSS Positioning Methods

I've been a bit confused by positioning, so I took this opportunity to delve a little deeper. For full disclosure - I adapted the content of this page from learnlayout.com, the most straight forward approach to this I've seen so far. I tried to reproduce the content and positioning from that page within this blog post. So, here we go!

STATIC POSITIONING

static is the default value. An element with position: static; is not positioned in any special way. A static element is said to be not positioned and an element with its position set to anything else is said to be positioned.

/* CSS code for static positioning */

.static {
position: static;
}

RELATIVE 1

relative behaves the same as static unless you add some extra properties.

RELATIVE 2

Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

/* CSS code for Relative positioning */

.relative1 {
position: relative;
}

.relative2 {
position: relative;
top: -4.2em;
left: 6em;
}

RELATIVE3

This element is relatively-positioned. If this element was position: static; its absolutely-positioned child would escape and would be positioned relative to the document body.

ABSOLUTE

This element is absolutely-positioned. It's positioned relative to its parent. Got that? It's nested inside RELATIVE3

/* CSS code for ABSOLUTE nested within RELATIVE positioning */

.relative3 {
position: relative;
}

.absolute {
position: absolute;
top: 12em;
right: 0;
width: 50%;
}

FIXED POSITION

And you probably noticed this pesky guy to the bottom right. Fixed positioning. Oooh yeah.

/* CSS code for FIXED positioning */

.fixed {
position: fixed;
bottom: 0;
right: 0;
width: 24%;
}

Why is JavaScript so popular in web development?

I googled the above question, and got some delightful responses. Recent-ish articles seem, on the whole, very enthusiastic about JavaScript, with titles such THE RISE AND RISE OF JAVASCRIPT, Why JavaScript Will Become The Dominant Programming Language Of The Enterprise, and Will JavaScript take over the programming world?.
Not bad.


As a newbie dev (full disclosure) I gleaned some very interesting tidbits about JS. It turns out that JS, until the last 5 years or so, was much maligned by experienced devs. Originally developed to add interactivity to web pages, it was engineered to be very lightweight, easy on bandwidth and the browser. This means JS inherently has limited data structures, and syntactically, it's less sweet than Ruby or Python.


JS was designed to allow asynchronous rendering of content in the browser. Meaning, JS lets the browser dynamically display content in response to events and user input. Thanks to JS, the page doesn't display flat and lifeless, but responds to all sorts of input.


JS is in all browsers - meaning it's a common language that every computer uses. And its ubiquity has spurned on some really exciting growth in directions likely not anticipated by its creators.


External libraries, like jQuery, simplified syntax and readability and added functionality.


JS has grown to be a server side language as well, via technologies like node.js. SO, the same language can function on both the client and the server side, which can speed interactions and data requests. And, it's also a super fast database laguage, via MongoDB.


Additionally - several mobile technologies use JS as the system framework. Which allows web developers to leverage their skills, and work as mobile devs without having to learn additional languages.


That's cool.


I don't really understand how it all works yet, but jeez.
That's cool.


There seems to be alot of speculation and anticipation in these articles about just how BIG JS can, and should, get. Will it dominate all sectors of development?

I don't know. I'm a newb.


BUT - I'm certainly left with the lingering question, WHY ARE WE LEARNING RUBY ON RAILS? If JS is so flexible across platforms, and is super intrinsic to web content, shouldn't we be focusing there?


Of course, I really have no idea. And thus far, I do feel that Ruby has a much lower threshold for entry for someone like me. I appreciate that. But maybe JS is the more flexible method in the long term.

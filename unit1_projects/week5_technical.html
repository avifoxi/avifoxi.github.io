(Un)real world applications for classes.


The world can be a very scary place. And with the aid of Ruby Classes, we can model the interactions of real (or not so real) objects. When we create a Class in Ruby, we are creating a template for a new Ruby object.
Say, for example, a SCARY MONSTER!!!


So - in keeping with the DBC ethic of embracing our fears in hopes of overcoming them, Welcome to the Class of SCARY MONSTERS!!
First we will define our class, and define what an initialized object of the Scary Monster class will contain.


class Scary_monster
def initialize(name, scariest_feature, speed, strength, alive=true) @name = name @scariest_feature = scariest_feature @speed = speed @strength = strength @alive = alive end

Notice that we'll initialize an object with 5 parameters, which will make for some SCARY interactions. When we instantiate a new Scary Monster, we provide 4 parameters (the 5th is default assigned). And these parameters become the INSTANCE VARIABLES common to EVERY object of this class. We designate instance variables with @ before the variable text.
Every Scary Monster will have a:

  • Name
  • Scariest Feature
  • Speed rating (statistically calculated between 1 - 10)
  • Strength rating (also scientifically calculated to a value between 1 and 10)
  • And a default alive variable of true

Sooo.... What do scary monsters do all day ?


Well... firstly, they SCARE.
So, we'd better provide a class method for our Scary Monsters do this, or they'll be very unfulfilled scary monsters.

def scare puts "BOO!! \n#{@name} comes outa nowhere and terrifies the bejeeezus outa you \nwith its #{@scariest_feature}!" end

Whoa! That is scary!
Notice how we make use of the instance variables. When a Scary_Monster calls the scare function, it's individual instance methods are inserted to the curly braces, for a personalized fright. Boo.
What else do Scary Monsters do ?
THEY ATTACK !!
And that is very scary.
very.
scary.

Here's where things also get a little complicated. We need 2 different objects of the same class to interact within the same method. We will need to be careful with instance variables, and create some additional methods to be used within the attack method. But don't be SCARED!


First off - how do we know which Scary Monster would win in a fight ?
We have to calculate the Whoop Quotient.
Obviously.

def whoop_quotient @speed * @strength end

We will also need to access a Scary Monster's name within the Attack method.

def name @name end

And if things don't go as planned for one of the monsters, we'll need a kill function, to change the @alive variable to false. As an interesting aside that I discovered in writing this blog post - you cannot delete an object once instantiated in Ruby! So - I suppose this is good news if you're a Scary Monster thats been whooped. And once whooped, rather than deleting we will simply assign the @alive instance variable to false.

def kill @alive = false end

Now we are ready for an ATTACK METHOD!!

BWAAHAAHAAHAA!!!!
Observe.

def attack(scary_monster) puts "#{@name} attacks #{scary_monster.name}!" if self.whoop_quotient > scary_monster.whoop_quotient puts "#{@name} whoops #{scary_monster.name}'s behind." scary_monster.kill else puts "OH NO! #{@name} is outclassed and makes a run for it!!! " end end


WAIT!!!
HOLD YOUR HORSES DUDE!!


This is scary.
We are in the Scary Monster Class.
We have a method 'attack' that an object of Scary_Monster can call, which accepts a parameter of scary_monster.

WHAT ??

ok - one object of Scary_Monster will call attack. In this method, the monster that calls the attack may reference its own instance variables. And when referring to itself in the method - we reference it as 'self'.

What does the parameter scary_monster point to?
It points to a DIFFERENT SCARY MONSTER that is the object of the attack. And to access this second object's instance variables, we have to use name and whoop quotient methods defined above.

Finally- our complete Class definition is closed, so let's be sure to end it.

end

Now - all the pieces are ready to play!
Let's make some Scary Monsters and MAKE 'EM FIGHT!!


boogeyman = Scary_monster.new('The Boogeyman', 'Shadow Lurk of Terror', 5, 3)
frankenstein = Scary_monster.new('Frankenstein', 'Cadaver Breath', 2, 8)
frankenstein.scare
puts ' '
boogeyman.scare
puts ' '
boogeyman.attack(frankenstein)
frankenstein.attack(boogeyman)
